# 每日灰豚数据对标账号笔记导出

## When to use
当用户需要每天从灰豚数据（xhs.huitun.com）的"我的收藏"页面自动导出对标账号的**新增笔记明细数据**时触发。典型用户指令：
- "执行今日导出"
- "帮我导出今天的灰豚数据"
- "每日对标账号抽取"

## Runtime requirements
- Browser login required: yes（需已登录灰豚数据账号）
- Sandbox required: no
- MCP required: no
- Page script required: no
- User local folder: optional（如挂载了本地目录则自动保存到指定路径）

## Capability routing
全程使用浏览器GUI操作：
1. **浏览器操作**：导航、点击、输入、弹窗处理、下载管理
2. **文件整理**：若用户挂载了本地目录，记录文件应存放位置；否则告知用户手动移动文件

## Workflow（正确流程 — 已验证）

### Step 1：打开我的收藏页面
- 导航至 `https://xhs.huitun.com/#/collect`
- 确认页面加载完成，看到"收藏达人"标签页和数据概览卡片

### Step 2：读取今日新增笔记数并点击"查看"
- 从页面顶部**数据概览卡片**中读取"新增笔记"字段值（例如：345条）
- 记录该数值为 `note_count`
- 获取当天日期，格式化为 `YYYYMMDD`（例如：20260625）
- **关键步骤**：点击"新增笔记"数字旁的红色 **"查看"** 链接
- 此操作会弹出**笔记信息对话框**，显示所有新增笔记的明细列表

### Step 3：在笔记列表页点击导出
- 在弹出的笔记信息对话框中，找到右上角的 **"导出"** 按钮
- 点击该按钮（注意：是在笔记列表弹窗内的导出，不是达人列表页面的导出）

### Step 4：填写导出参数
在弹出的"数据导出"对话框中：

1. **导出名称**：`{YYYYMMDD}-对标账号笔记-{note_count}条`
   - 示例：`20260625-对标账号笔记-345条`
   - 注意名称中是"对标账号**笔记**"（不是"对标账号"）
   - 字符限制：不超过20个字符

2. **导出条数**：修改结束值为 `{note_count}`
   - 与新增笔记数一致
   - 确认预估导出条数显示正确

3. 点击 **"确定"** 按钮

### Step 5：确认导出成功
- 等待绿色对勾 + "导出成功"提示弹窗
- 弹窗文字："数据导出成功，请前往导出中心下载数据"
- 点击 **"确定"** 关闭提示

### Step 6：前往导出中心下载
- 页面会自动跳转至 `https://xhs.huitun.com/#/tools/export`（或手动导航）
- 在历史导出记录列表中，**第一行**就是刚导出的文件
- 文件名格式：`{YYYYMMDD}-对标账号笔记-{note_count}_331534951`
- 状态应为："导出成功"
- 点击该行的 **下载按钮**（⬇️图标）

### Step 7：确认下载完成
- 使用 `list_downloads` 确认文件下载完成（state=completed）
- 记录下载文件名和保存路径

### Step 8：文件归档（可选，依赖本地目录挂载）
如果用户本地目录可用（`/mnt/local` 存在）：
```
目标路径：/mnt/local/灰豚数据导出/{YYYYMMDD}/
```
如果本地目录不可用：
- 告知用户文件在浏览器默认下载目录
- 提供手动移动的目标路径：`D:\自媒体创业项目\灰豚数据导出\{YYYYMMDD}\`

## Output contract

每次执行完成后必须向用户报告以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 执行日期 | 导出当天日期 | 2026-06-25 |
| 新增笔记数 | 从页面读取的实际数量 | 345条 |
| 导出文件名 | 按规则生成的完整名称 | 20260625-对标账号笔记-345条_331534951 |
| 导出条数 | 实际导出的数据行数 | 345条 |
| 导出类型 | 笔记明细数据（非达人列表） | 笔记明细 |
| 下载状态 | completed / in_progress / failed | completed |
| 文件位置 | 实际保存路径或待手动移动路径 | 浏览器下载目录 / D:\...\20260625\ |

## 关键注意事项（踩坑记录）

### ❌ 常见错误
1. **不要直接在"收藏达人"标签页点导出** → 那样导出的是达人列表数据，不是笔记明细
2. **必须先点"查看"进入笔记列表** → 这是获取笔记数据的唯一正确入口
3. **文件名必须是"对标账号笔记"** → 不是"对标账号"，少了"笔记"二字是错误命名
4. **导出入口是笔记弹窗内的导出按钮** → 不是达人列表页右上角的导出按钮

### ✅ 正确流程特征
- 页面路径：我的收藏 → 收藏达人 → 点击"新增笔记"旁的"查看" → 笔记信息弹窗 → 导出
- 导出内容：每条笔记的标题、发布者、时间、标签等明细信息
- 文件名包含"笔记"二字
- 条数与"新增笔记"数字一致

## Failure handling

| 场景 | 处理方式 |
|------|---------|
| 未登录灰豚数据 | 提示用户先登录，停止执行 |
| 页面显示验证码弹窗 | 尝试处理滑块验证码，失败则提示用户手动处理后重试 |
| 新增笔记数为0 | 告知用户今日无新数据，询问是否仍要导出 |
| "查看"链接不可见 | 截图反馈当前页面状态，排查原因 |
| 笔记信息弹窗未出现 | 检查是否点到了正确的"查看"链接（uid通常在数据概览卡片区） |
| 导出按钮不可用 | 截图反馈当前弹窗状态，排查原因 |
| 下载失败 | 重试一次下载，仍失败则报告错误 |
| 导出额度不足 | 提示剩余额度，建议调整导出条数或升级套餐 |
| 本地目录不可用 | 文件下载到默认目录，提供手动归档指引 |

## 重要提醒
- 灰豚数据每日导出上限 **10,000条**
- 导出名称限制 **20个字符以内**（注意"对标账号笔记-XXX条"格式的字符数）
- 导出记录保留最近30天
- 如遇页面改版，以实际页面元素为准灵活调整选择器
- **核心原则：先"查看"后"导出"，顺序不能反**
